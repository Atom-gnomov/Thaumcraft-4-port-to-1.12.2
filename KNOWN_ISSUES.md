# Известные проблемы

Список открытых багов/недоделок порта Thaumcraft 4 → 1.12.2 на текущий момент.
Закрытые пункты переносятся в `CHANGELOG.md` при исправлении, отсюда удаляются.

## Открыто

- **Перепроверить старые баг-репорты на новой базе (после ребейза на FOREVA)** —
  плиты, банки, тигель, кристаллы, цвета флюидов. Часть могла уже решиться сама
  собой при ребейзе, часть — нет; нужен прогон по каждому пункту в игре.
- **Нитор (Nitor): неправильная текстура** — должен быть красный огонёк, сейчас
  не тот.
- **Таумометр: брошенный предмет должен лежать горизонтально**, а не стоять
  вертикально, как обычный дропнутый айтем.
- **Броня Fortress + пустотная роба: рендерится розовым** (классический признак
  отсутствующей/неверно указанной текстуры или модели) — нужна отладка в игре с
  логами.
- **Волшебный биом (Magical Forest): неверная цветовая гамма** — например,
  изумрудная трава и т.п. должны быть тонированы иначе.
- **Исследования (Thaumonomicon) для новых блоков/предметов** — часть контента,
  добавленного при портировании (в т.ч. 1.8–1.12.2 ванильные аспекты), не имеет
  research-записей. Запланировано после релиза 1.0.

## Известные архитектурные ограничения (не баги, а осознанные компромиссы)

- Просканированные предметы игрока хранятся как необратимый хэш
  (`ScanManager.generateItemHash`) — обратный поиск `ItemStack` по хэшу работает
  только через фоновый кэш `MappingThread`/`GuiResearchRecipe.cache`, который
  строится один раз за сессию и может не успеть заполниться в первые секунды
  после захода в мир.
